Manage Rules

Section #3: Rules

This is the most important process because what is set up here can be enabled to force users to submit expedited reports for AEs they are documenting in the application.  The application will determine, based on the rules configured here, what the user should do.

 

Rules can have one condition, or more than one condition, and they will be based on “IF” statements.

 

Conditions

 

There will be a minimum of 4 drop down boxes and you will need to select an option from each box.  The drop down boxes are for the Domain Object, Field, Operator, and Value.

♦    Domain Object - From the drop down list you need to choose one of the following four options.

o  Adverse Event - this rule will pertain strictly to AEs being reported

o  Study - this rule will pertain to studies

o  Report Definitions - this rule will pertain only to particular report definitions already created.

o  Course - this rule will pertain only courses added to the caAERS application.

Depending on what selection you make from the drop down box for Domain Object, the next drop down box for Field is going to have different criteria.  Each option below will be based on what selection is chosen from the drop down.

♦    Adverse Event - Field options

o  Grade - This will allow you to set the grade level that will require an expedited report.

o  Hospitalization - This will allow you to set what level of hospitalization will require an expedited report.

o  Expected - This will allow you to set the rule dependent on whether or not the AE was expected to occur.

o  Attribution - This will allow you to set the attribution of the AE to the study as a reason for enforcing the user to enter an expedited report.

♦    Study - Field options

o  Phase - This will allow you to set up the rule based on what trial phase the study is in.

o  IND Holder - This will allow you to set a rule for an IND that an expedited report must be submitted for an AE incurred due to the IND used in the study.

o  Therapy - This will allow you to set a rule for atherapy that an expedited report must be submitted for an AE incurred due to the therapy used in the study.

♦    Report Definition - Field options

o  Name - This will say if a certain report definition is being used to send a report for an AE than it should also be sent to the people assigned to the report definition that will be set up later in this section.

♦    Course - Field options

o  Treatment Assignment Code -

 

The next set of drop down boxes for Operator are going to be very similar no matter what option you selected from Field.  The Operators can be one of the following:

♦    Equal To

♦    Not Equal To

♦    Greater Than or Equal To

♦    Less Than or Equal To

Each operator is referencing the Value that will be specified in the next drop down box.  Depending on what selection you make from the drop down box for Field, the next drop down box for Value is going to have different criteria.  Each option below will be based on what selection is chosen from the Field drop down.

♦    Adverse Event Grade

o  0: Normal

o  1: Mild

o  2: Moderate

o  3: Severe

o  4: Life-Threatening or disabling

o  5: Death

♦    Adverse Event Hospitalization

o  None

o  Hospitalization

o  Pro-longed Hospitalization

♦    Adverse Event Expected

o  Yes

o  No

♦    Adverse Event Attribution

o  Unrelated

o  Unlikely

o  Possible

o  Probable

o  Definite

♦    Study Phase

o  Phase I Trial

o  Phase I/II Trial

o  Phase II Trial

o  Phase III Trial

o  Phase IV Trial

o  Phase V Trial

o  Phase 0 Trial

o  Pilot

♦    Study IND Holder

o  This will include a list of IND holders added to caAERS by your administrator

♦    Study Therapy

o  Agent

o  Radiation

o  Surgery

o  Device

o  Behavioral

♦    Report Definition Name

o  This will include a list of Report Definitions that have been added to caAERS previously

♦    Course Treatment Assignment Code

o  This will include a list of Treatment Assignment Codes added to caAERS previously.

 

Actions

 

Once the conditions are set for the rule, you need to define what action is going to take place when a user encounters the rule.  The Actions are relegated to the Report Definitions created in a different section of caAERS.  Select the report from the drop down box you want to assign to this rule.

 

Click Continue when the Rules section is entered completely. If any data fields are entered incorrectly, an appropriate error message will appear after Continue is clicked. Error messages will describe data problems in specific fields. Correct the problems, and click Continue again.  The data will then be saved to the database.

Click Reset tab to clear all form fields.

Click the Back button to return to the previous page.  Data entered will be maintained in the web page memory, but not be saved to the database.
