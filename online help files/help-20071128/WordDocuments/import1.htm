Administration Module

Import

Studies and participants that have been created in other applications and other databases can be imported into caAERS so they do not have to be manually entered in the application one by one.  The studies or participants in the other application/database need to be put into a valid XML file for uploading into caAERS.  Studies and participants can not be in the same file.

 

Once the file is created you may go to the Import tab of the Administration Module.  For example files please go to https://gforge.nci.nih.gov/svnroot/caaersappdev/docs/import/.

 

Selecting either the Import Study/Protocol button or the Import Participant button will provide the Browse button on the page.  Navigate to the directory containing the XML file and double-click on it.  If the file passes all the validations you will be successful.  Depending on the size of the file it could take anywhere from minutes to hours to complete.         

 

After importing studies you may go to the Studies module to search for all the studies you imported.  You will know you were successful when you see the imported studies in the list.  The same steps can be taken for participants to make sure the import was successful.

 

 

 

Click Save when the Import section is entered completely.  The data will then be saved to the database.

Click Reset tab to clear all form fields.

Click the Back button to return to the previous page.  Data entered will be maintained in the web page memory, but not be saved to the database.
