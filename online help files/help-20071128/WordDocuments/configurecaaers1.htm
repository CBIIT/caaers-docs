Administration Module

Configure caAERS

caAERS is built around email notifications being sent to the appropriate people if certain criteria is met.  In order for the application to work properly you will need to provide the correct information for your SMTP mail server.  Otherwise, the notifications will not be sent and the application’s full capabilities will not be utilized.

 

          In order for the notifications to be sent correctly to the appropriate people, the following four fields must be entered correctly:

 

♦    SMTP Server – This must be the address of your outgoing mail server.  For example, smtp.gmail.com

♦    SMTP Port – The default port is 25, but if you are utilizing a different port to send outgoing mail enter it here.

♦    SMTP Password – The password used to send outgoing mail.

♦    SMTP User Name – The user name used to send outgoing mail

 

The final field is not required to send emails, but you can use it to specify the email address the messages will be sent from.  This does not have to be a valid email address, but it can be used so the people being notified of any AEs can quickly recognize where the email came from and what it is about.

 

Other parts of the application can be configured as well.  For example, if your organization is using the Patient Study Calendar application, it can be hooked into caAERS and the AEs can be placed on the calendar.  Also, if you are interested in developing, you can choose to show the debugging information. 

 

 

Click Save when the Configure caAERS section has been updated as needed.  The data will then be saved to the database.

Click Reset tab to clear all form fields.

Click the Back button to return to the previous page.  Data entered will be maintained in the web page memory, but not be saved to the database.
