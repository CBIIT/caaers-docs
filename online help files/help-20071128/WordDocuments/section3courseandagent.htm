Adverse Events Module > Enter Expedited Report

Section #3: Course and Agent

Treatment information on the course and agent the participant was subjected to during the AE is necessary to see the steps taken in helping the participant recover from the AE. 

♦    Treatment assignment code

o  You will see a list of codes in the drop-down box to select from.  These codes represent different drugs used in the study and will help identify if there is a pattern in AE’s being reported.  If the treatment assignment is appropriate for the participant you must select the radio button.

♦    Description of treatment assignment or dose level

o  In the text box enter a brief description of the doses taken of the agent and to add other information that details the treatment

♦    Enter a description of treatment assignment or dose level

o  If there are no values in the above mentioned Treatment assignment code drop-down box, or none of the codes match for the participant, enter details for the treatment they did receive. Otherwise, leave this field blank.

♦    Start date of first course

o  Enter the beginning date of the course (or cycle) that is associated with the primary Adverse Event or death on this report. Some protocols do not have defined courses or cycles, such as those with daily administration of an agent.  For these protocols “Start date of first course” is the date the patient last received the agent prior to experiencing the adverse event or death.

♦    Start date of course associated with expedited report

o  When this expedited report began, on what date did the current course begin.

♦    Course number on which event occurred

o  Enter the course (or cycle) associated with the Adverse Event(s) or death for this report. Some protocols do not have defined courses or cycles, such as those with daily administration of an agent.  For these protocols “Course number on which event occurred” is to be entered as 1 (one).

♦    Total number of courses to date

o  Enter the number of courses the participant has gone through, if applicable.

 

Once the information for the course has been supplied, information for the Study agent can be entered if necessary.  Click the “Add a study agent” button to display a form on the page where information about the agent can be provided.  Multiple agents can be supplied in the expedited report.

♦    Study Agent

o  The list includes all agents in all Treatment Assignment Codes (TAC’s) for the protocol selected.  Select the appropriate agent.

o  Each agent included on a treatment arm or dose level (TAC) must be addressed one at a time in this section of the report.

♦    Unit of measure

o  Select the unit of measure for the dose level taken from the drop down box.

♦    Date last administered

o  This field is mandatory for all investigational agents.  When an investigational agent is selected from the List of Values, enter the date that agent was last administered.

♦    Administration delay

o   Indicate numeric value and unit of Measure (minutes, hours or days) of delay.

♦    Comments

o  If there was an administration delay, provide the reason(s) for the delay.

♦    Dose modified

o  If the dose was modified from the default amount usually given, select the check box.

♦    Modified dose

o   If “Dose modified?” is selected, enter any changes to the dose given for this course (cycle). If the agent was not given exactly as described in the TAC, select from the list.  Data including “per m2, per kg,” etc., is not to be entered in this field.  In the subsequent fields enter the agent units and route.  Adjustment information applies to dosages for the course on which the event occurred. 

 

 

Click Continue when the Course and Agent tab is entered completely or if there is no information applicable. If any data fields are entered incorrectly, an error message will appear after Continue is clicked. Error messages will describe any problems with the data.  Correct the problem(s) and select Continue again. The data will be saved to the database.

 

Click Reset tab to clear all form fields.

Click the Back button to return to the previous page.  Data entered will be maintained in the web page memory, but not be saved to the database.
