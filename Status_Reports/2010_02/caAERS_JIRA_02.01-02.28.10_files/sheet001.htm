caAERS CAAERS-3794 Need to change the way parent - child reports are recommended by the rules Improvement Open Major UNRESOLVED Srini Akkala Paul Baumgartner 2/23/2010 16:14 2/23/2010 16:14 Version 2.0.1 Version 2.5 Module 12: Rules / Automated Decision Support
Module 4: Adverse Event Reporting		   0   43200 43200   0%     Identified by RPCI Currently, when a parent report is submitted, all of it's children are automatically created. This means that you can never have a 24hr notification without all of the other child reports automatically being created. Since a parent can have multiple children, this also can result in multiple reports of the same type being created at the same time - a situation that the UI prevents. Instead of this implementation, we should make the following change:

When a child report is recommended, recommend the parent be completed first.

This change will allow you to also author the rules so that a 24hr notification can be recommended by itself without automatically starting all of the "hidden" child reports. Conversely, if we make this change, we will not be able to create a child report without first completing it's parent.

To summarize, the change is this:

Instead of automatically creating the children when a parent is recommended, do the opposite - create the parent when the child is recommended.
When both the parent and child reports are recommended, create the parent and then once submitted, create the child.
If only the parent is recommended, create only the parent and after it is submitted, do not create the child.
The rules will be able to be reauthored such that only the child reports are recommended since the parent report will always be automatically created.		   0% 0%   43200 43200
